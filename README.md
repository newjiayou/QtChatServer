# QmlChatApp - 高可靠即时通讯客户端

基于 **Qt 6 (QML & C++)** 开发的即时通讯软件，采用逻辑与表现层完全解耦的架构，实现了自定义二进制协议通信、高可靠连接管理及本地数据持久化。

## 🚀 核心技术亮点

### 1. 自定义二进制通讯协议
为了解决 TCP 通信中的**粘包与半包**问题，本项目自主设计了应用层协议：
- **报文格式**：`TotalLength (4字节)` + `MessageType (2字节)` + `DataBody (JSON/Binary)`。
- **解析策略**：采用缓冲区（Buffer）预读机制，只有当缓冲区字节数满足总长度字段时才进行拆包，确保数据解析 100% 准确。

### 2. 高可靠连接管理 (断线重连与心跳)
- **指数退避算法 (Exponential Backoff)**：当网络断开时，自动触发重连机制，尝试间隔随失败次数呈指数级增长（1s, 2s, 4s... 封顶 60s），有效避免服务器瞬时压力过大。
- **双向心跳机制**：通过定时发送 Ping 包及超时检测定时器（30s），识别“掉线但未断开”的死链接，并强制触发重连。

### 3. C++ 驱动的高性能数据模型
- **Model-View 架构**：继承 `QAbstractListModel` 实现 `chatmodel`，通过重写 `roleNames` 将 C++ 结构体直接映射到 QML 视图。
- **性能优化**：支持万级聊天记录异步加载，通过信号槽机制实现 UI 的局部刷新，确保界面滚动无卡顿。

### 4. 数据持久化
- 集成 **SQLite** 数据库，实现聊天历史、用户配置的本地存储。
- 支持加载历史记录至 Model，实现丝滑的翻页查看体验。

## 🛠 技术栈
- **前端**：QML, Qt Quick Controls 2
- **后端**：C++ 17, Qt Network (QTcpSocket), Qt SQL (SQLite)
- **构建工具**：CMake


## 📥 编译与运行
1. 环境要求：Qt 6.2+ (编译器建议 MinGW 或 MSVC)
2. 使用 Qt Creator 打开 `CMakeLists.txt`。
3. 编译并运行。默认连接本地服务器 `127.0.0.1:12345`。
